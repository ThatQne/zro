# zro — Personal Minimalist Browser

Fast, lightweight Tauri v2 browser for Windows. Custom WebView2 compositing, modular Rust architecture, AI agent with semantic memory.

## Features

### Core
- **UI-on-top compositing** — overlays (sidebar, panels, dropdown) render genuinely *over* the page webview, never shift the viewport
- **One-row top bar** — logo, navigation, URL, actions, window controls in a single 40px row
- **Per-tab webviews** — independent page instances; inactive tabs parked off-screen (zero reflow on switch)
- **Instant resize** — synchronous GDI region tracking per window event; rounded corners work on every site
- **Rounded corners + inset** — 12px corner margin, 10px content corner radius (live-computed)

### Navigation & SPA
- **SPA tracking** — YouTube video→video, Gmail, etc. never show stale URLs (SourceChanged→page-navigated event)
- **Back/forward** — native WebView2 GoBack/CanGoBack with URL fallback
- **Smart URL bar** — history + open tabs + Google suggestions; Ctrl+L/Ctrl+E to focus
- **Search engines** — Google / DuckDuckGo / Bing (Settings panel)
- **Keyboard shortcuts** — Ctrl+T new · Ctrl+W close · Ctrl+Shift+T reopen · Ctrl+H history · Ctrl+Tab cycle · Ctrl+, settings · Ctrl+J downloads · F5 reload · Ctrl+± zoom

### Sidebar & Tabs
- **Collapsible sidebar** — hover-expand, drag-reorder tabs + folders (dnd-kit), context menus
- **Folder system** — organize tabs by color/icon; batch ops (close, move, duplicate)
- **Multi-select** — Shift/Ctrl + click; bulk actions in sidebar context menu
- **Tab persistence** — restore session on restart (zustand partialize + localStorage)

### Downloads & Tools
- **Downloads manager** — track active/completed with pause/resume UI
- **History panel** — search with title/URL; clear all (with confirmation)
- **Built-in cookie editor** — list/copy/add/delete cookies for active page (Header / JSON / netscape.txt formats)
- **Incognito mode** — per-session toggle; no history logging
- **Stats panel** — page metrics (in dev)

### AI Assistant
- **Multi-provider** — Ollama (local) · OpenAI-compatible SSE · mz-code (local CLI agent)
- **Tool loop** — read_page, click, fill, navigate, scroll, search_history, get_cookies
- **Semantic memory** — nomic-embed-text embeddings + turbovec index; persistent facts DB
- **Markdown rendering** — code blocks, bold, italics, links in chat
- **Auto-focus** — new blank tabs focus URL bar; Ctrl+Alt+A toggles AI panel

### Performance
- **Instant.page prefetch** — 65ms hover intent, mousedown trigger (same-origin only)
- **Lazy webview creation** — tabs are parsed, webviews created on first activation
- **Live instrumentation** — `[nav] loaded in Xms` per page (browser console)
- **Profile persistence** — WebView2 profile in `%LOCALAPPDATA%/com.zro.browser/EBWebView` (565MB, survives restarts)

### Dev & Customization
- **Modular Rust** — `browser/{mod,layout,overlay,tabs,cookies,…}.rs` + `agent/{mod,eval,semantic,…}.rs`; glob re-exports keep command paths clean
- **Zustand stores** — tabs.ts, ai.ts, downloads.ts, overlays.ts (frontend state synced to Rust via IPC)
- **Source code** — Tauri v2.11.2, React 18, Framer Motion, lucide-react, dnd-kit, TypeScript

Opens the app window with hot-reload enabled. Edit `.tsx` files and changes appear live; Rust changes require a restart.

- **Event inspection** — `log_js` command funnels JS errors to Rust console
- **Network** — `preview_network` in Claude Code preview tools

## Build & Installer

### Release Build
```bash
npm run tauri build
```

Outputs:
- `src-tauri/target/release/zro.exe` — standalone binary
- `src-tauri/target/release/bundle/msi/` — Windows installer (MSI)
- `src-tauri/target/release/bundle/nsis/` — NSIS installer

### MSI Installer
Auto-generated by Tauri. Signed builds require a certificate; unsigned are dev-signed.

**To install:**
1. Run the MSI from the bundle folder
2. Or `npm run tauri build -- --target msi`

**To package manually:**
```bash
# Generate installer without signing
npm run tauri build --config productName="zro" bundler=nsis
```

### Distribute
Copy the generated `.exe` (portable) or `.msi` (installer) to users.

For automatic updates, configure `src-tauri/tauri.conf.json`:
```json
"updater": {
  "active": true,
  "endpoint": "https://your-domain.com/updates"
}
```

## Architecture Notes

### Render Loop
1. UI webview (main, 40px bar + overlays) raised to `HWND_TOP`
2. GDI region = full window − rounded content hole + overlay rects (RGN_DIFF + RGN_OR)
3. Page webview renders in the hole; clicks in hole fall through to page

### Tab Lifecycle
1. New tab → register in store, webview created on first switch
2. Navigate → `navigate_tab` command or `page-loaded` event
3. Switch → `sync_bounds` repositions active webview, parks others at x=−32000
4. Close → remove from store, destroy webview

### State Sync
- **Frontend** → **Rust**: JS `invoke()` for native operations (navigate, get_cookies, etc.)
- **Rust** → **Frontend**: Tauri events (page-loaded, page-navigated, page-title, shortcut, etc.)
- **Persistence**: Zustand `partialize` saves tabs/history/settings to localStorage; Rust has persistent WebView2 profile

### AI Agent (Rust-side)
- `agent/tools.rs` — tool definitions (read_page, click, fill, navigate, search_history, get_cookies)
- Providers: Ollama (HTTP) / OpenAI-compatible SSE / mz-code (spawn child CLI)
- Semantic memory: `index_page` on load-finish (deferred 2.5s to avoid layout thrashing)
- Tool loop: fetch response, parse tool_call, execute, stream result back to frontend

## Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| Pages don't load | Missing WebView2 runtime | Install WebView2 (included on Win11) |
| Overlays render under page | Old build | `npm run tauri build --target x86_64-pc-windows-msvc` |
| "Add account" Gmail button dead | `additionalBrowserArgs` set | Never set additionalBrowserArgs in tauri.conf.json |
| Ctrl+± / Ctrl+scroll don't zoom | wry default | Already enabled in code; if not working, check `.zoom_hotkeys_enabled(true)` in tabs.rs |
| AI stuck / no response | Ollama/API down | Check provider is running; try different provider in Settings |
| History doubled | StrictMode race (fixed) | Already fixed in this version |
| Video won't fullscreen | Element fullscreen handler | Check console for ContainsFullScreenElementChanged errors |

## Known Limitations

- **Custom renderer** — WebView2 only for now; Servo path behind experimental flag on separate branch
- **Extensions** — WebView2 doesn't support Chrome Web Store CRX
- **Multi-window** — single browser window; popup `window.open` becomes new tabs (by design)
- **Sync** — no built-in cloud sync; use browser sync extension if needed
